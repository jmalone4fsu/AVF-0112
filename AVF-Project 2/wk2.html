Question 1: What are some ways you could change your HTML on your Demo app to be more accessible?

Make sure all img tags include a descriptive alt tag. Add html5 markup for captions and subtitles to all my videos. Use custom controls for my videos that make control easier.

Question 2: What are some common frustrations with Screen Readers and how to fix them?

One common frustration with them is that a lot of sites doen't use markup that makes them Reader accessible. Proper coding could easily fix that. Also, the screen Reader is dependent on the device that it is running on, so making special keys or easier options on the device would help.

Question 3: List 2 devices (and features) that are made specifically to support accessibility?

Mountbatten Brailler - (http://www.mountbattenbrailler.com) is a brailler writer, a notetaker, and a brailler embosser, all in one, for blind and visually impaired.

iPhone - the iphone is build with the most accessibility options for the hearing and visually impaired. It offers Voiceover screen reader, can connect via bluetooth to wireless braille displays, has Zoom feature for visually challenged, white-on-black contrast for better visibility, Speak Selection will speak highlighted text and read formatting options, Visual Voicemail for those that can not hear, and audible - visual - and vibrating alerts.

Question 4: List 2 mobile apps made specifically for accessibility?

Qwitter Client - Qwitter is an accessible Twitter client designed for access by the blind, that integrates between Twitter and a screen reader.

Touchscreen Braille Writer - Android - virtual braille keyboard.
